Introduction

Flexbox is a CSS3 layout mode, which is used to accommodate design layouts on different screen sizes and display devices. Many web developers find flexbox easier to use than block or grid layouts.

Flexbox is considered easier to use for many applications because it does not use floats and the flex container's margins do not collapse with the margin's of it's contents. Another reason why many designers find flexbox easier than older layout models is the widths and heights of flexboxes vary to adapt to the display space. This helps hold the lower down elements in place, like when you need a sticky footer, which can be a tricky task to deal with when using other methods.

Another benefit of flexbox is, it takes into consideration whether you want div's to accrue to the right or the bottom of the page. The display order of flexbox elements is also independent of their order in the source code.

The characteristics of flexbox ensure popular layouts are much easier to implement with simpler and cleaner code!

Parent Properties

The properties below are to be applied to flexbox containers. They will affect all direct children of the container element.

display

This defines a flex container to be either block or inline.

.container {
display: flex | inline-flex;
}

flex-direction

This establishes the main-axis of the container, which defines the direction flex items are placed in the flex container. Flex items primarily layout in horizontal rows or vertical columns. By default flex-direction is set to row.

.container {
flex-direction: row | row-reverse | column | column-reverse;
}

flex-wrap

Flex items will all try to fit on one row. If needed, you change the default behavior and allow items to wrap.

.container {
flex-wrap: nowrap | wrap |wrap-reverse;
}
  • nowrap: all flex items will be on one line.
  • wrap: flex items will wrap on to multiple lines.
  • wrap-reverse: flex items will wrap on to mulitple lines from bottom to top.

flex-flow

Flex-flow condenses flex-direction and flex-wrap in one property. This defines the flex container's main and cross axes. Default is row nowrap.

.container {
flex-flow: <'flex-direction'> <'flex-wrap'>;
}

justify-content

This defines the alignment of items along the main axis. It distributes extra free space when the items are inflexible or have reached their maximum size. It will also exert some control over the alignment of items when they overflow to mulitple lines.

.container {
justify-content: flex-start | flex-end | center | space-between | space-around;
}
  • flex-start (default): items are packed towards the start of the line.
  • flex-end: items are packed towards the end of the line.
  • center: items are packed towards the center of the line.
  • space-between: first item is at the start of the line, last item is at the end of the line and other items are evenly distributed along the line.
  • space-around: items are evenly distributed along the line with equal space around them.

align-items

This property is like justify-content, but for items on the cross-axis instead of main-axis.

.container {
align-items: flex-start | flex-end | center | baseline |stretch;
}
  • flex-start: items are placed on the cross start line
  • flex-end: items are placed on the cross end line
  • center: items are centered on the cross-axis
  • baseline: items are aligned so that their baselines align
  • stretch (default): stretch items to fill the container (min-width/max-width is still respected.

align-content

This aligns a flex container's lines when there is extra space in the cross-axis, similar to how justify-content aligns individual items within the main-axis. This property has no effect when there is only one line of flex items.

.container {
align-content: flex-start |flex-end | center | space-between | space-around | stretch;
}
  • flex-start: lines packed to the start of the container
  • flex-end: lines packed to the end of the container
  • center: lines packed to the center of the container
  • space-between:the first line is at the start of the container, the last line is at the end of the container and the rest are evenly distributed
  • space-around: lines are evenly distributed with equal space around each line
  • stretch: lines stretch to take up the remaining space
Child Properties

order

Flex items, by default, are laid out in source code order. The order property controls the order in which the items will be displayed.

.item {
order: <integer>;
}

flex-grow

This gives flex items the ability to grow, if needed. Flex-grow accepts a number that serves as a proportion that dictates how much space the item should take up inside the flex container. Negative numbers are invalid.

.item {
flex-grow: <number>;
}

flex-shrink

This gives flex items the abiltity to shrink, if needed. Negative numbers are invalid.

.item {
flex-shrink: <number>;
}

flex-basis

This defines the size of an item before the remaining space is distributed. It can be a length or the keyword 'auto'. If flex-basis is set to 0, the extra space around the content is not factored in. When set to auto, the extra space is distributed based on its flex-grow value.

.item {
flex-basis: <length> | auto;
}

flex

This property is shorthand for when you need to combine flex-grow, flex-shrink and flex-basis. Default is 0 1 auto. Flex-shrink and flex-basis are optional, but it is recommended to use this shorthand property rather than setting indiviual properties because the shorthand sets the other values intelligently.

.item {
flex: none | [ <'flex-grow'> <'flex-shrink'>? || <'flex-basis'>} ];
}

align-self

This overwrites the default alignment (or the alignment specified by align-items) for individual flex items.

.item {
align-self: auto | flex-start | flex-end | center | baseline | stretch;
}
Useless Properties

The following properties have no affect on flex items:

  • float
  • clear
  • vertical-align
